  • 12. Oct – 18. Oct
    Peter: Kick-off meeting, definovanie vízie, rozdelenie rolí, koordinácia motivačného listu a predstavenia tímu.
    Roman: Vytvorenie projektovej webstránky (kostra), sekcie tím, motivačný list, nahratie na hosting.
    Anton: Príprava hostingu a prostredia pre projektovú stránku a budúcu aplikáciu.
    Boris: Úvodný návrh vizuálnej identity, moodboard a UX nápady pre helpdesk.
    Gabriel: Úvodný návrh dátového modelu a diskusia o štruktúre databázy.
  • 19. Oct – 25. Oct
    Peter: Návrh modulov systému (tickets, users, statusy) a prvý návrh API endpointov.
    Boris: UX flow pre základné scenáre (vytvorenie, zobrazenie a správa ticketov), tvorba prvých wireframov.
    Gabriel: Detailnejší ER diagram a návrh databázovej schémy.
    Anton: Vytvorenie základného backend projektu a API skeleton.
    Roman: Rozšírenie projektovej stránky o harmonogram a dokumentáciu.
  • 26. Oct – 1. Nov
    Peter: Spolupráca na definovaní dátových modelov a rozdelenie ďalších vývojových úloh.
    Boris: UI prototyp hlavného dashboardu v Figma a iterácia podľa feedbacku tímu.
    Gabriel: Implementácia databázy (MySQL/PostgreSQL) a napojenie na backend.
    Anton: Implementácia prvých API endpointov (login, zoznam ticketov, detail ticketu).
    Roman: Začiatok tvorby aplikačnej stránky – predstavenie aplikácie AlfaDesk.
  • 2. Nov – 8. Nov
    Peter: Kontrola prvého funkčného prepojenia backendu a databázy, nastavenie priorít pre prvú verziu.
    Boris: Návrh a doladenie formulárov pre vytvorenie a úpravu ticketu.
    Gabriel: Testovanie spojenia API ↔ DB, úpravy dotazov a dátových typov.
    Anton: Nasadenie backendu na server (staging prostredie), príprava Docker konfigurácie.
    Roman: Rozšírenie aplikačnej stránky o sekciu „Funkcionalita“ a priestor pre budúce demo.
  • 9. Nov – 15. Nov
    Peter: Zavedenie Git/GitHub workflow na riadenie a zdieľanie kódu (branchovanie, PR, úlohy), koordinácia integrácie frontend–backend.
    Boris: Finalizácia základného vzhľadu dashboardu a obrazovky ticketu podľa možností implementácie.
    Gabriel: Refaktor databázovej štruktúry podľa výsledkov testovania, doplnenie indexov.
    Anton: Rozšírenie API o filtrovanie, zmenu statusov a komentáre, príprava CI pipeline.
    Roman: Začiatok systematického testovania prvých funkcií (vytváranie a zobrazenie ticketov), report chýb.
  • 16. Nov – 22. Nov
    Peter: Revizia plánu do prezentácie 20. 12., stanovenie must-have funkcií a rozdelenie úloh.
    Boris: Doladenie responzívneho UI a UX pre hlavné obrazovky.
    Gabriel: Finalizácia DB schémy pre verziu na prezentáciu, príprava testovacích dát.
    Anton: Stabilizácia backendu, logovanie chýb a základný monitoring.
    Roman: Príprava a spustenie testovacích scenárov pre API a UI, zapisovanie chýb do Git-u.
  • 23. Nov – 29. Nov
    Peter: Spolupráca na návrhu scenára živého dema a prezentácie architektúry.
    Boris: Doplnenie UX detailov (stavy, chybové hlášky, loader) pre kľúčové funkcie.
    Gabriel: Kontrola dátovej integrity pri hlavných scenároch, úpravy constraintov.
    Anton: Príprava skriptov na rýchle nasadenie prezentovanej verzie a zálohovanie.
    Roman: Retest opravených chýb a kontrola end-to-end scenárov.
  • 30. Nov – 6. Dec
    Peter: Kontrola stavu kľúčových funkcií a definovanie finálneho rozsahu na prezentáciu.
    Boris: Polishing UI – farebná konzistencia, ikonky a vizuálne detaily.
    Gabriel: Ladenie výkonu SQL dotazov (paginácia, filtrovanie).
    Anton: Optimalizácia odozvy API a riešenie prípadných problémov s hostingom.
    Roman: Príprava sekcie na projektovom webe s odkazom na demo a dokumentáciu.
  • 7. Dec – 13. Dec
    Peter: Nácvik prezentácie – rozdelenie rolí, čo kto ukazuje a vysvetľuje.
    Boris: Úpravy UI podľa posledného feedbacku z testovania a nácviku prezentácie.
    Gabriel: Príprava ukážkových dát pre demo (typické tickety, stavy, kategórie).
    Anton: Príprava „freeze“ verzie aplikácie (tag v Git-e, finálny deploy).
    Roman: Kompletné regresné testovanie a final bug report.
  • 14. Dec – 20. Dec
    Peter: Finalizácia prezentácie (slidy, príbeh, časovanie), komunikácia s vyučujúcimi.
    Boris: Príprava screenshotov a vizuálnych podkladov do prezentácie.
    Gabriel: Posledná kontrola databázy a pripravených skriptov (reset, inicializácia dát).
    Anton: Monitoring behu aplikácie počas prezentácie, pripravené záložné riešenie.
    Roman: Kontrola všetkých odkazov na webovej stránke (demo, PDF, prezentácie) a podpora počas prezentácie.
