ALFADESK Helpdesk

Hlavný projekt tímu ALFASYSTEMS. Nižšie je priebežný harmonogram podľa týždňov.

ALFADESK – harmonogram (12. okt → 20. dec)

  • 12. Oct – 18. Oct
    Peter: Kick‑off meeting, predstavenie vízie a základná architektúra.
    Gabriel: Návrh DB schémy (entitno‑vzťahový model) a pripojenie k backendu.
    Roman: Nastavenie repozitára a CI, kostra webu a testovacie scenáre.
    Anton: Príprava hostingu, domény a Docker prostredia.
    Boris: Wireframy helpdesku a UI komponenty pre dashboard.
  • 19. Oct – 25. Oct
    Peter: Definícia dátových tokov medzi modulmi, návrh MQTT štruktúr.
    Gabriel: Vytvorenie základnej databázy v MySQL a testovanie spojenia s API.
    Roman: Implementácia základných endpointov API a testovacia verzia UI.
    Anton: Nastavenie Docker Compose a správa kontajnerov backend/frontend.
    Boris: Úvodný dizajn dashboardu v Figma, farebná paleta a typografia.
  • 26. Oct – 1. Nov
    Peter: Implementácia MQTT komunikácie a logiky spracovania udalostí.
    Gabriel: Optimalizácia databázy, doplnenie indexov a validácie dát.
    Roman: Rozšírenie testov, validácia API a prvý deploy na interný server.
    Anton: Automatizácia build procesu, CI/CD pipeline.
    Boris: Responzívny layout pre mobilné zariadenia a testovanie UI.
  • 2. Nov – 8. Nov
    Peter: Integrácia modulov IoT a webového backendu.
    Gabriel: Ladenie dátových modelov, doplnenie relácií.
    Roman: Prvá verzia dashboardu so základnými funkciami.
    Anton: Monitorovanie servera a logov, správa domény.
    Boris: Zlepšenie UX toku pre riešenie ticketov.
  • 9. Nov – 15. Nov
    Peter: Analýza výkonu a úprava architektúry pre stabilitu.
    Gabriel: Testovanie integrácie s API a refaktor DB vrstvy.
    Roman: Rozšírenie testov a oprava chýb z prvej revízie.
    Anton: Zálohovanie systému a nastavenie alertov.
    Boris: Doplnenie vizuálnych prvkov pre správu úloh.

Pozn.: ďalšie týždne budú doplnené priebežne po kontrole pokroku.
